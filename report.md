## 4. Fixes Needed

### Fix #1 â€” `skip_nonce_check` should be `false` in production  *(High, pre-production)*
**Location:** `supabase/config.toml:333`
**Issue:** `skip_nonce_check = true` is set for the Google OAuth provider. This is necessary for local sign-in testing but **disables the nonce check that prevents login CSRF and authorization-code interception**. If this same config is used in production, it is a real security downgrade.
**Fix:** Set `skip_nonce_check = false` once the Google OAuth flow is confirmed working end-to-end in production. Keep `true` only in a local-dev-only config profile if needed.
**Action:** Verify the prod Supabase project's Google provider has nonce checking enabled.

### Fix #2 â€” Avoid `git add .` ; stage explicitly (do not commit `dev-dist/`)  *(Low, at commit time)*
**Location:** Untracked files include `dev-dist/registerSW.js` (a Vite PWA dev artifact).
**Issue:** A blanket `git add .` would stage the dev artifact and possibly other generated files. `dev-dist/` should be gitignored (it is regenerated by `vite-plugin-pwa` in dev mode).
**Fix:** Stage files explicitly, or confirm `dev-dist/` is in `.gitignore` before adding. Only commit source, config, tests, and the new docs.
**Action:** Verify `.gitignore` contains `dev-dist` (it currently ignores `dist` and `dist/sw.js`, `dist/workbox-*.js` â€” confirm `dev-dist` is covered).

### Fix #3 â€” MFA enrollment UI still pending  *(Medium, post-commit)*
**Location:** Noted in `REMEDIATION_CHECKLIST.md:35`
**Issue:** The MFA *RLS enforcement* migration exists (`20240104000000_enforce_mfa_rls.sql`), but the **frontend enrollment flow** (`supabase.auth.mfa.enroll()` / `verify()`) is not implemented. Admins currently cannot enroll an MFA factor, so the RLS policy may lock them out or be effectively bypassed.
**Fix:** Build the MFA enrollment UI (QR code + TOTP verify) and gate it behind `is_admin = true` in the `profiles` table.
**Action:** Add to backlog; do not deploy admin features to production until this exists.

### Fix #4 â€” Captcha on auth forms still pending  *(Medium, post-commit)*
**Location:** Noted in `REMEDIATION_CHECKLIST.md:36`
**Issue:** The rate-limit Edge Function template exists, but **Cloudflare Turnstile / hCaptcha is not integrated** into the signup/login forms. Brute-force protection currently relies only on Supabase's default rate limits.
**Fix:** Obtain a Turnstile key, enable `[auth.captcha]` in `config.toml`, and add the captcha widget to `AuthPage.tsx`.
**Action:** Add to backlog.
